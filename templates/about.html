What this website does

This site is intended to be a resource for cooperative gaming. The other major statistics websites for both Dota and League focus almost entirely on "what can I as a player do to beat the other team". I started this project to try and show that there is still usefulness and market value in getting players to think "cooperative" instead of only "competitive". This site is definitely a work in progress! Here's how to read it:

The first click on the hero / champion grid selects the first hero and displays which items to buy in what order along with their winrate and best friends - regardless of their teammates or opponents. Suggested item completion order is top to bottom.

The second click on the grid shows both solo and paired items for both heroes. The selected two can be displayed as either Foes or Friends (on opposite teams and on the same team) using the selector switch below the hero / champion grid. You can see how the items change from solo to paired by looking for differences between the columns. You can think of the Teammates / Opponents view as 2v0 and 1v1.

I’ve developed the recommendation algorithm by doing some machine learning over about 600,000 matches for each game. The algorithm is based on various types of random forest classification, and I’m improving it all the time. One weakness I’m working to address is the multiple recommendations for boots, especially with Boots of Travel in DotA matches. Be sure to check back - the recommendation algorithm should improve every few days.

You may notice that the winrates aren’t sequential with buy order - that’s intentional! If the buy order was based only on winrate then you’d always get the lategame items first. As I improve the algorithm then you’ll see better and better build orders, which may or may not keep the same sequencing.

The newest patch for DotA hasn’t been out long enough for me to grab enough matches for good statistics - It’s currently displaying data for 6.87 and I’ll update it as soon as I can get enough matches from the Valve API.

Examples

For League, I recommend checking out Annie / Viktor as friends and foes - without being informed about support or carry roles my algorithm picks up on the winning trend of Annie buying support items when paired with Viktor and carry items when fighting him.

For Dota, I recommend Weaver / Ogre Magi - when they're opponents my algorithm suggests Linkens earlier since it deals with Ogre. When they're on the same team the build changes back towards 'normal'

Better recommendations

I'm tweaking the algorithm all the time, so the build order is getting better. If you notice an issue I'd like to hear about it so that I can update around it.

Rationale

ESports are great, and I notice that both Dota and League focus on marketing them to the exclusion of almost everything else. I think that turns away a potential user base of people who want to share the games they love with their friends. This site is being designed to take out the "build" part of Dota and League, so that when new players get dragged in by their friends they have a lower barrier to entry.
